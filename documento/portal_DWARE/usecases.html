borrar_activoID Caso de Uso: 010

Nombre Caso de Uso: Borrar Activo.

Fecha creación: Noviembre 24 de 2008
creado por : Jairo Lavado Hernámdez.

Fecha modificación:
Modificado por :

Objetivo:
Eliminar el registro de activos del sistema.

Actores:
Gestor RIA.

Precondiciones:
1. El usuario debe haber iniciado una session.
2. El sistema debe estar en la pagina principal del modulo de Activos, con un listado de activos.

Poscondiciones:
Se modifico los datos del registro de un activo.

Flujo Normal:
1.El usuario ubica el activo en la lista y selecciona en la lista frente al activo, la opcion borrar Activo.
2.El sistema muestra un mensaje de confirmación de borrado.
3.El usuario Acepta el borrado del activo.
4.El sistema borra el registro en la base de datos y muestra la pagina principal del modulo Activos, mostrando una lista general de activos.

Flujos Alternativos:

* El usuario seleccioan cancela en el mensaje de confirmación.

4.El sistema muestra la pagina principal del modulo Activos, mostrando una lista general de activos, sin borrar registros.

Resultado:
Se Borra el registro de un activo de la base de datos del sistema.

Fecha creación: Noviembre 24 de 2008

Objetivo:
Modificar los datos de los registros de Activos.

Actores:
Gestor RIA.

Precondiciones:
1. El usuario debe haber iniciado una session.
2. El sistema debe estar en la pagina principal del modulo de Activos, con un listado de activos.

Poscondiciones:
Se modifico los datos del registro de un activo.

Flujo Normal:
1.El usuario ubica el activo en la lista y selecciona en la lista frente al activo, la opcion editar Activo.
2.El sistema muestra un formulario, con los datos del activo a Editar.
3.El usuario Modifica el tipo o el nivel o los datos del activo y presiona el boton aceptar.
4.El sistema actualiza el registro en la base de datos y muestra la pagina principal del modulo Activos, mostrando una lista general de activos.

Flujos Alternativos:

* El tipo de activo no existe en el sistema.

3.El usuario selecciona en el menu agregar tipo de activo(caso de uso crear tipo).
4.El sistema muestra el formulario para ingresar tipo.
5.El usuario ingresa los datos del tipo de activo y presiona el boton aceptar.
6.El sistema guarda el registro del tipo en la base de datos, muestra el formulario con los datos del activo y el tipo que se ingreso seleccionado.
7.El modifica los datos del activo y presiona el boton aceptar.

* El nivel de activo no existe en el sistema.

3.El usuario selecciona en el menu agregar nivel de activo(caso de uso crear nivel).
4.El sistema muestra el formulario para ingresar nivel.
5.El usuario ingresa los datos del nivel del activo y presiona el boton aceptar.
6.El sistema guarda el registro del nivel en la base de datos, muestra el formulario con los datos del activo ingresados, y el nivel que se ingreso seleccionado.
7.El usuario modifica los datos del activo y presiona el boton aceptar.
8.El sistema guarda el registro en la base de datos y muestra la pagina principal del modulo Activos, mostrando una lista general de activos.

* El usuario decide cancelar la modificación del registro.
- El sistema muestra una lista de activos, en la pagina principal de modulos de activos.


Resultado:
Se modifican los datos un activo.

El usuario oprime el boton cancelar en le formulario editar.

6.El sistema muestra la pagina principal del modulo administrar noticia, mostrando una lista noticias, sin modificar el registro de la noticia.

Resultado:
Se edita un registro de noticia que se puede visualizar en la pagina principal del portal dependeindo del tipo de noticia.

Fecha creación: Noviembre 24 de 2008
creado por : Jairo Lavado Hernámdez.

Fecha modificación:
Modificado por :

Objetivo:
Ingresar un nuevo Activo en el sistema.

Actores:
Gestor RIA.

Precondiciones:
1. El usuario debe haber iniciado una session.
2. El sistema debe estar en la pagina principal del modulo de administración.

Poscondiciones:
Se ingresa un activo al sistema, para ser valorado.

Flujo Normal:
1.El usuario selecciona en el menu, Activos.
2.El sistema va a la pagina principal del modulo Activos.
3.El usuario selecciona en el menu, la opcion ingresar Activo.
4.El sistema muestra un formulario, para ingresar un nuevo Activo; mostrando en el menu desplegable Nuevo Activo.
5.El usuario selecciona el tipo, el nivel, ingresa el nombre, la descripción, caracteristicas del activo y presiona el boton aceptar.
6.El sistema guarda el registro en la base de datos y muestra la pagina principal del modulo Activos, mostrando una lista general de activos, con el activo que se acaba de ingresar

Flujos Alternativos:

* El activo ya existe en el sistema.

4.El sistema muestra un formulario, para ingresar un nuevo Activo; mostrando en el menu desplegable una lista con los nombres de los activos registrados.
5.El usuario selecciona el nombre del activo en la lista desplegable.
6.El sistema actualiza el formulario, con el nombre y la descripción del activo.
7.El usuario selecciona el tipo, el nivel e ingresa las caracteristicas del activo y presiona el boton aceptar.
8.El sistema guarda el registro en la base de datos y muestra la pagina principal del modulo Activos, mostrando una lista general de activos, con el activo que se acaba de ingresar

* El tipo de activo no existe en el sistema.

5.El usuario selecciona en el menu agregar tipo de activo(caso de uso crear tipo).
6.El sistema muestra el formulario para ingresar tipo.
7.El usuario ingresa los datos del tipo de activo y presiona el boton aceptar.
8.El sistema guarda el registro del tipo en la base de datos, muestra el formulario con los datos del activo ingresados, y el tipo que se ingreso seleccionado.
9.El usuario selecciona el nivel e ingresa los datos del activo y presiona el boton aceptar.
10.El sistema guarda el registro en la base de datos y muestra la pagina principal del modulo Activos, mostrando una lista general de activos, con el activo que se acaba de ingresar

* El nivel de activo no existe en el sistema.

5.El usuario selecciona en el menu agregar nivel de activo(caso de uso crear nivel).
6.El sistema muestra el formulario para ingresar nivel.
7.El usuario ingresa los datos del nivel del activo y presiona el boton aceptar.
8.El sistema guarda el registro del nivel en la base de datos, muestra el formulario con los datos del activo ingresados, y el nivel que se ingreso seleccionado.
9.El usuario selecciona ingresa los datos del activo y presiona el boton aceptar.
10.El sistema guarda el registro en la base de datos y muestra la pagina principal del modulo Activos, mostrando una lista general de activos, con el activo que se acaba de ingresar

Se guarda un registro de un activo descrito, caracterizado, al que se puede evaluar.
