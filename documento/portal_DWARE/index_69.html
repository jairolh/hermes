editar_activouse caseID Caso de Uso: 008

Nombre Caso de Uso: Editar Activo.

Fecha creación: Noviembre 24 de 2008
creado por : Jairo Lavado Hernámdez.

Fecha modificación:
Modificado por :

Objetivo:
Modificar los datos de los registros de Activos.

Actores:
Gestor RIA.

Precondiciones:
1. El usuario debe haber iniciado una session.
2. El sistema debe estar en la pagina principal del modulo de Activos, con un listado de activos.

Poscondiciones:
Se modifico los datos del registro de un activo.

Flujo Normal:
1.El usuario ubica el activo en la lista y selecciona en la lista frente al activo, la opcion editar Activo.
2.El sistema muestra un formulario, con los datos del activo a Editar.
3.El usuario Modifica el tipo o el nivel o los datos del activo y presiona el boton aceptar.
4.El sistema actualiza el registro en la base de datos y muestra la pagina principal del modulo Activos, mostrando una lista general de activos.

Flujos Alternativos:

* El tipo de activo no existe en el sistema.

3.El usuario selecciona en el menu agregar tipo de activo(caso de uso crear tipo).
4.El sistema muestra el formulario para ingresar tipo.
5.El usuario ingresa los datos del tipo de activo y presiona el boton aceptar.
6.El sistema guarda el registro del tipo en la base de datos, muestra el formulario con los datos del activo y el tipo que se ingreso seleccionado.
7.El modifica los datos del activo y presiona el boton aceptar.
8.El sistema guarda el registro en la base de datos y muestra la pagina principal del modulo Activos, mostrando una lista general de activos.

* El nivel de activo no existe en el sistema.

3.El usuario selecciona en el menu agregar nivel de activo(caso de uso crear nivel).
4.El sistema muestra el formulario para ingresar nivel.
5.El usuario ingresa los datos del nivel del activo y presiona el boton aceptar.
6.El sistema guarda el registro del nivel en la base de datos, muestra el formulario con los datos del activo ingresados, y el nivel que se ingreso seleccionado.
7.El usuario modifica los datos del activo y presiona el boton aceptar.
8.El sistema guarda el registro en la base de datos y muestra la pagina principal del modulo Activos, mostrando una lista general de activos.

* El usuario decide cancelar la modificación del registro.
- El sistema muestra una lista de activos, en la pagina principal de modulos de activos.


Resultado:
Se modifican los datos un activo.

editar_noticiause caseID Caso de Uso: 004

Nombre Caso de Uso: editar noticia.

Objetivo: Modificar los datos de uan noticia publicada.

Fecha creación: Noviembre 14 de 2008
creado por :

Fecha modificación:
Modificado por :
Actores:
Administrador_sistema.

Precondiciones:
1. El usuario debe haber iniciado una session.
2. El sistema debe estar en la pagina principal del modulo de administración de noticias.

Poscondiciones:
Los datos de la noticia que se ha editado se podra ver en la pagina principal del portal.

Flujo Normal:
1.El usuario selecciona en el menu, la opcion del tipo de noticia a editar.
2.El sistema muestra una lista de noticias del tipo que el usuario selecciono en la pagina principal de administrar noticias.
3.El usuario selecciona la opcion editar, junto a la noticia a editar.
4.El sistema muestra un formulario, con los datos de la noticia.
5.El usuario modifica el tipo de noticia, el titulo y/o el cuerpo de la noticia y presiona el boton aceptar.
6.El sistema modifica el registro en la base de datos y muestra la pagina principal del modulo administrar noticia, mostrando una lista con la noticia que se acaba de modificar

Flujos Alternativos:

El usuario oprime el boton cancelar en le formulario editar.

6.El sistema muestra la pagina principal del modulo administrar noticia, mostrando una lista noticias, sin modificar el registro de la noticia.

Resultado:
Se edita un registro de noticia que se puede visualizar en la pagina principal del portal dependeindo del tipo de noticia.

eliminar_noticiause caseID Caso de Uso: 005

Nombre Caso de Uso: eliminar noticia.

Objetivo: Eliminar el registro de una noticia publicada.

Fecha creación: Noviembre 14 de 2008
creado por :

Fecha modificación:
Modificado por :

Actores:
Administrador sistema.

Precondiciones:
1. El usuario debe haber iniciado una session.
2. El sistema debe estar en la pagina principal del modulo de administración de noticias.

Poscondiciones:
Los datos de la noticia que se ha editado se podra ver en la pagina principal del portal.

Flujo Normal:
1.El usuario selecciona en el menu, la opcion del tipo de noticia a eliminar.
2.El sistema muestra una lista de noticias del tipo que el usuario selecciono en la pagina principal de administrar noticias.
3.El usuario selecciona la opcion eliminar, junto a la noticia a editar.
4.El sistema muestra un mesaje de confirmación para eliminar la noticia.
5.El usuario oprime la opcion SI.
6.El sistema busca el registro de la noticia en la base de datos y lo borra, luego muestra la pagina principal del modulo administrar noticia, mostrando la lista de noticias, exepto la que se acabo de eliminar.

Flujos Alternativos:

El usuario selecciona la opcion NO en el mensaje de confirmación.

6.El sistema muestra la pagina principal del modulo administrar noticia, mostrando una lista noticias, sin eliminar el registro de la noticia.

Resultado:
Se borra un registro de noticia que ya no se podra visualizar en la pagina principal del portal.
