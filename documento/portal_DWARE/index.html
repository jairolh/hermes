ID Caso de Uso: 001

Nombre Caso de Uso:Iniciar sesión

Fecha creación: Noviembre 14 de 2008
creado por :

Fecha modificación:
Modificado por :

Objetivo: Acceder al sistema

Actores
Administrador sistema, Gestor_RIA, Gestor_IIA

Precondiciones:
El usuario debe estar registrado en el portal

Poscondiciones:
Ingreso del usuario al sistema, según privilegios.

Flujo Normal:
1.El portal en su pagina principal, muestra el formulario de ingreso
2.El usuario digita el nombre y la clave de usuario
3.El sistema identifica tipo de usuario.
4.El sistema desbloquea los módulos al que el usuario puede acceder.

Flujos Alternativos:
Clave o nombre de usuario erradas

1.El usuario digita mal el nombre y/o la clave de usuario
2.El sistema muestra el formulario para ingresar nuevamente el nombre y clave de usuario, con el error que ocurrió.

Resultado:
Usuario interactuando con el sistema

Fecha creación: Noviembre 14 de 2008
creado por :

Fecha modificación:
Modificado por :

Actores
Administrador sistema, Gestor_RIA, Gestor_IIA

terminar interacción del usuario con el sistema

Modificado por :
Actores:
Administrador_sistema.

Precondiciones:

2. El sistema debe estar en la pagina principal del modulo de administración de noticias.

Los datos de la noticia que se ha editado se podra ver en la pagina principal del portal.

1.4.1.12 Use Case ingresar_activo

ID Caso de Uso: 007

Nombre Caso de Uso: Ingresa Activo.

Fecha creación: Noviembre 24 de 2008
creado por : Jairo Lavado Hernámdez.

Fecha modificación:
Modificado por :

Objetivo:
Ingresar un nuevo Activo en el sistema.

Actores:
Gestor RIA.

Precondiciones:
1. El usuario debe haber iniciado una session.
2. El sistema debe estar en la pagina principal del modulo de administración.

Poscondiciones:
Se ingresa un activo al sistema, para ser valorado.

Flujo Normal:
1.El usuario selecciona en el menu, Activos.
2.El sistema va a la pagina principal del modulo Activos.
3.El usuario selecciona en el menu, la opcion ingresar Activo.
4.El sistema muestra un formulario, para ingresar un nuevo Activo; mostrando en el menu desplegable Nuevo Activo.
5.El usuario selecciona el tipo, el nivel, ingresa el nombre, la descripción, caracteristicas del activo y presiona el boton aceptar.
6.El sistema guarda el registro en la base de datos y muestra la pagina principal del modulo Activos, mostrando una lista general de activos, con el activo que se acaba de ingresar

Flujos Alternativos:

* El activo ya existe en el sistema.

4.El sistema muestra un formulario, para ingresar un nuevo Activo; mostrando en el menu desplegable una lista con los nombres de los activos registrados.
5.El usuario selecciona el nombre del activo en la lista desplegable.
6.El sistema actualiza el formulario, con el nombre y la descripción del activo.
7.El usuario selecciona el tipo, el nivel e ingresa las caracteristicas del activo y presiona el boton aceptar.
8.El sistema guarda el registro en la base de datos y muestra la pagina principal del modulo Activos, mostrando una lista general de activos, con el activo que se acaba de ingresar

* El tipo de activo no existe en el sistema.

5.El usuario selecciona en el menu agregar tipo de activo(caso de uso crear tipo).
6.El sistema muestra el formulario para ingresar tipo.
7.El usuario ingresa los datos del tipo de activo y presiona el boton aceptar.
8.El sistema guarda el registro del tipo en la base de datos, muestra el formulario con los datos del activo ingresados, y el tipo que se ingreso seleccionado.
9.El usuario selecciona el nivel e ingresa los datos del activo y presiona el boton aceptar.
10.El sistema guarda el registro en la base de datos y muestra la pagina principal del modulo Activos, mostrando una lista general de activos, con el activo que se acaba de ingresar

* El nivel de activo no existe en el sistema.

5.El usuario selecciona en el menu agregar nivel de activo(caso de uso crear nivel).
6.El sistema muestra el formulario para ingresar nivel.
7.El usuario ingresa los datos del nivel del activo y presiona el boton aceptar.
8.El sistema guarda el registro del nivel en la base de datos, muestra el formulario con los datos del activo ingresados, y el nivel que se ingreso seleccionado.
9.El usuario selecciona ingresa los datos del activo y presiona el boton aceptar.
10.El sistema guarda el registro en la base de datos y muestra la pagina principal del modulo Activos, mostrando una lista general de activos, con el activo que se acaba de ingresar

Resultado:
Se guarda un registro de un activo descrito, caracterizado, al que se puede evaluar.
